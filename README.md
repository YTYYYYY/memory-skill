# Memory Skill - 设计与架构文档

## 概述

Memory Skill 是 OpenClaw 系统中的一个核心管理技能，专门用于管理长期复杂任务的生命周期。它通过结构化的记忆系统，帮助 AI Agent 在多个会话之间保持上下文连续性。

## 设计目标

1. **跨会话记忆** - 任务状态和上下文在会话之间持久化
2. **自动备份** - 每次操作后自动同步到 MinIO 存储
3. **快速检索** - 通过关键词、标签、时间范围快速定位任务
4. **模板化** - 常见任务类型提供预设模板

## 核心组件

### 1. SKILL.md - 技能描述文件

定义技能的元信息和使用场景：
- 技能名称：memory
- 使用场景：管理长期复杂任务、跨会话上下文保持

### 2. memory.sh - 核心脚本

Bash 脚本，提供完整的命令行接口：

| 命令 | 功能 |
|------|------|
| `templates` | 列出可用模板 |
| `init-task` | 创建任务（支持模板） |
| `save-step` | 保存任务进度 |
| `save-decision` | 保存关键决策 |
| `update-phase` | 更新任务阶段 |
| `search` | 搜索任务 |
| `index` | 查看索引 |
| `backup` | 备份到 MinIO |
| `restore` | 从 MinIO 恢复 |

### 3. 索引系统 (index.json)

维护所有任务的元数据索引，支持：
- 按关键词搜索
- 按标签筛选
- 按时间范围查询
- 按阶段筛选

### 4. MinIO 备份机制

三层备份模式：
- **sync** - 每次操作立即同步
- **incremental** - 仅备份修改的文件
- **lazy** - 手动备份

## 使用示例

```bash
# 1. 创建任务（带模板，自动检索历史）
memory.sh init-task task-001 bugfix "登录"

# 2. 保存进度
memory.sh save-step task-001 1 "已复现问题"

# 3. 保存决策
memory.sh save-decision task-001 "使用方案A" "性能更好"

# 4. 搜索
memory.sh search "登录" "" "2026-03-01" "2026-03-31"

# 5. 查看索引
memory.sh index

# 6. 手动备份
memory.sh backup

# 7. 恢复
memory.sh restore
```

## 环境变量

| 变量 | 说明 | 默认值 |
|------|------|--------|
| AGENT_NAME | Agent 名称（自动检测） | default |
| AGENT_BACKUP_MODE | 备份模式 | sync |

## 扩展性

- **自定义模板**：支持添加、编辑、删除自定义模板
- **备份模式**：可根据性能需求选择不同的备份策略
- **Agent 隔离**：每个 Agent 的数据独立存储在 MinIO
